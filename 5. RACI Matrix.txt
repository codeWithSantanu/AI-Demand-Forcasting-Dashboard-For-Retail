---

🧠 A. INTRODUCTION (FOR REPORT)

In team-based software development, clear task allocation is essential to ensure efficiency and accountability. To manage responsibilities effectively, we used the RACI Matrix model, which defines roles for each task in terms of Responsibility, Accountability, Consultation, and Information sharing.

This approach ensures that:

Every task has a clear owner
No duplication of work occurs
Team coordination is maintained
Project delivery stays on schedule

---

📌 B. WHAT IS RACI?

| Term                | Meaning                              |
| ------------------- | ------------------------------------ |
| R (Responsible) | Person who performs the task         |
| A (Accountable) | Person responsible for final outcome |
| C (Consulted)   | People whose input is taken          |
| I (Informed)    | People kept updated                  |

⚠️ E. IMPORTANT RULES (VERY IMPORTANT)

👉 From HIT guideline:

Each task must have exactly ONE Responsible (R)
Team Lead is Accountable (A) for all tasks
No task should have multiple “R”
