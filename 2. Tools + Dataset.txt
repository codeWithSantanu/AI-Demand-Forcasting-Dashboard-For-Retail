🛠️ 2. TOOLS + DATASET (FULL DETAIL — REPORT + VIVA + IMPLEMENTATION)

---

🧱 A. INTRODUCTION (FOR REPORT)

To develop the SahayakAI – Smart Government Scheme Navigator, we selected a set of tools and dataset strategies that ensure simplicity, efficiency, and feasibility within a 1-week timeline. The system is designed using a full-stack architecture with a lightweight dataset and optional AI integration.

The focus was on choosing tools that:

Are easy to implement
Support rapid development
Provide reliable performance
Are suitable for MVP development

---

🛠️ B. TOOL STACK (DETAILED)

🔹 1. FRONTEND TECHNOLOGY

✅ Tool: React.js

📌 Purpose:

Build user interface (UI)
Handle form inputs and display results

💡 Why React?

Component-based architecture
Fast rendering
Easy integration with backend APIs

📍 Used For:

Input form (user details)
Results dashboard (schemes + score)

---

🔹 2. BACKEND TECHNOLOGY

✅ Tool: Node.js + Express.js

📌 Purpose:

Handle API requests
Process eligibility logic
Return results

💡 Why Node.js?

Lightweight and fast
Easy JSON handling
Ideal for REST APIs

📍 Used For:

`/check` API
Eligibility filtering
Score calculation

---

🔹 3. DATABASE / DATA STORAGE

✅ Tool: JSON Dataset (Local Storage)

📌 Purpose:

Store scheme details

💡 Why JSON?

No setup required
Easy to read and modify
Perfect for small dataset (MVP)

---

🔹 4. AI TOOL (OPTIONAL)

✅ Tool: **OpenAI API / Claude API

📌 Purpose:

Convert scheme details into simple language

💡 Why Optional?

Core logic does NOT depend on AI
Keeps system stable if API fails

✅ Backend: Render

Host application online

---

📦 DATASET STRUCTURE

Each scheme includes:

```json
{
  "name": "Scheme Name",
  "age_min": 18,
  "max_income": 300000,
  "category": ["low_income"],
  "documents": ["aadhaar", "income_certificate"],
  "benefit": "Scheme benefit",
  "link": "official website"
}
```

---

📌 DATA FIELDS EXPLANATION

| Field      | Description                  |
| ---------- | ---------------------------- |
| name       | Name of scheme               |
| age_min    | Minimum eligible age         |
| max_income | Income limit                 |
| category   | Target user group            |
| documents  | Required documents           |
| benefit    | What user gets               |
| link       | Official application website |

⚡ E. DATASET CREATION APPROACH

Steps followed:

1. Identify major government schemes
2. Extract eligibility criteria
3. Normalize data into structured format
4. Store in JSON file
5. Validate links and fields

---

🧠 F. WHY THIS DATASET APPROACH IS STRONG

✔ Easy to implement
✔ Fast processing
✔ Suitable for filtering logic
✔ Scalable (can add more schemes later)
